■ 案件名
建設業向けコンストラクション・マネジメントサービス デジタル化基盤構築

■ フェーズ
設計 / 構築：2024年1月 ～ 2025年3月（稼働率50%）
運用保守 / 追加開発：2025年4月 ～ 現在（稼働率30～100%）

■ 体制
開発メンバー10名

■ 役割
サブリード / クラウド基盤設計担当

■ 技術スタック
Azure（Application Gateway / Static Web Apps / Web Apps / SQL Managed Instance / Azure DNS）
Terraform
Azure DevOps（CI/CD）
Python（Django）
Vue.js

■ プロジェクト概要
建設業向けコンストラクション・マネジメントサービスのデジタル化に伴うクラウド基盤構築および継続的な運用保守・追加開発を担当。IaCおよびCI/CDにより標準化された基盤を構築し、運用フェーズでは監視・セキュリティ・バッチ基盤の整備を実施。

■ 担当業務（設計 / 構築フェーズ）
・Terraformによる3環境の新規IaC設計およびモジュール化
・Azureアーキテクチャ設計（Application Gateway / Web Apps / SQL Managed Instance）
・CI/CDパイプライン設計・構築（自動デプロイ化）
・設計支援、コードレビュー、基盤標準化

■ 担当業務（運用保守 / 追加開発フェーズ）
・Azure Log Analytics を用いたアプリケーションログ監視基盤の設計・構築
・Djangoミドルウェアにてリクエスト単位ログ（JSON）を出力し、KQLによる横断的な調査基盤を整備
  - エラー原因特定
  - ユーザー行動履歴の追跡
  - リクエスト単位のパフォーマンス分析を可能化
・RLS（行レベルセキュリティ）によるマルチテナント制御の設計・実装
・Azure Container Apps を利用した定期メール送信バッチ基盤の構築

■ 成果
・インフラのコード化による再現性の高い環境構築プロセスを確立
・手動デプロイを廃止し、自動リリース体制を実現
・監視/ログ基盤の整備により障害調査および運用効率を向上
・属人化を排除し、継続的に改善可能な運用基盤を確立
